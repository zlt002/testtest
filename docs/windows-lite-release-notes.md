# Windows Lite 更新说明

## 最近 10 次更新

- 页面代码库识别更稳了：优化了路由匹配和业务信号判断，避免只因为命中了站点 `host` 就把普通页面误判成业务页面。
- 页面代码库分析结果更聚焦了：现在会优先给出更紧凑的证据卡片、推荐命令和知识库定位建议，排查页面归属和接口线索更直接。
- 新增 DOCX 文档能力：现在支持围绕 Word 文档进行创建、编辑和分析，覆盖常见的文档结构、图片、表格等处理场景。
- 修复页面编辑按钮偶发不可点的问题：页面状态刷新过程中，编辑入口不再被错误禁用，切回编辑会更顺手。
- 文本选择体验更稳定：重构了文本高亮实现方式，减少选择态异常，同时优化了 Docx 导出时的图片处理表现。
- 对话气泡和文本交互做了细化优化：折叠展示更自然，子内容查看和文本选中反馈更清晰。
- 新增运行事件流存储与恢复机制：运行中断后更容易续上，相关预览资源路径解析能力也一起补齐了。
- 子代理状态展示更清楚：等待、活跃、异常等状态被合并到卡片标签里，摘要信息更集中，阅读负担更小。
- 修复了一批扩展和 backend 的失败用例，整体稳定性有所提升。
- 增强了 Plan 模式下的计划确认测试覆盖，涉及退出 Plan 模式和权限切换的交互会更可靠。

## 你能感知到的变化

- 如果你经常用页面代码库分析，这次版本对“识别是否真的是业务页面”会更谨慎，误判会更少。
- 如果你会做页面编辑、文本选择、内容折叠查看，这次交互细节和稳定性会比之前更顺。
- 如果你会用到运行恢复、预览资源定位、子代理状态查看，这次信息展示和恢复链路会更完整。
- 如果你需要处理 Word 文档，这次已经补上了一套新的 DOCX 能力支持。
