# Windows Lite 更新说明

## 本次更新

- 新增技能目录自检能力，可在管理页快速检查本地 `skills` 目录是否完整，并给出是否需要重新远端同步的提示。
- 优化远端同步恢复逻辑：当本地技能目录缺失或异常时，会自动触发重新同步，减少同步成功但本地能力不完整的问题。
- 优化首次同步与空目录场景处理：当本地目标目录还不存在时，不再因为备份步骤影响同步流程，初次落盘更顺滑。
- 管理工作台增强了失效技能/文件的容错处理，遇到已被删除或失效的能力项时会自动刷新列表并恢复到可继续操作的状态。
- 修复第三方模型配置里 `OpenAI 模型`、`OpenAI URL`、`Anthropic 模型`、`Anthropic URL` 输入框清空后会被默认值自动顶回的问题，现可稳定保存为空。
- 修复项目模型配置保存后的来源面板跳转问题：保存项目模型配置后，界面会继续停留在当前项目模型配置页，不会再被自动切回“用户级 Claude settings”。
- 修复官方网关地址识别兼容性，`.../api-sse-anthropic` 与 `.../api-sse-anthropic/v1` 现在都会被正确识别为官方配置，减少来源判断误差。
- 优化模型配置与空状态提示区的交互，失败场景下的操作按钮更聚焦，官方 Key 配置入口更紧凑清晰。
- 修复 Windows Lite 自动更新收尾体验：当更新包安装完成后，旧侧边栏会主动关闭并重新拉起，避免出现“更新说明已是新版本，但右侧侧边栏仍停留在旧版本界面”的情况。
- 优化本地预览资源的加载兼容性，跨上下文打开预览内容时更稳定，减少部分资源无法显示的问题。
- 调整聊天输入区与部分配置面板的文案/字号表现，提升 Windows Lite 下的阅读和填写体验。

## 升级提示

- 如果你发现技能列表为空、技能文件打不开，建议先到管理页执行一次“技能自检”，按提示决定是否重新同步。
- 如果你需要手动切换第三方模型或自定义网关地址，现在可以先清空原有值再重新填写，不会再被默认配置自动回填。
- 如果你刚保存项目模型配置，页面会继续留在项目模型配置表单里；右上角“当前生效”状态仍会单独显示真实运行来源，方便继续排查。
- 正常情况下，升级完成后旧侧边栏会自动关闭并恢复为新版本界面；如果浏览器拦截了自动恢复，仍可在扩展管理页手动重新加载一次扩展。
- 如果本次升级后仍提示模型或同步异常，优先使用界面里的“重新检查”入口，再决定是否重新安装或重新同步。
