# WebEdit Word

适用场景：当前会话需要在 WebEdit 在线文档里阅读、理解、改写、续写、插入或替换正文内容。

核心流程：

1. 先列工具：调用 `mcp__browser_extension__list_website_tools`，确认存在文档相关 `webedit_*` tools。
2. 先读取：优先使用可用的文档读取工具，例如正文、选区、光标上下文相关工具。
3. 再写入：只有在上下文已确认、工具真实可用时，才执行插入或替换。
4. 写后验证：再次读取正文或选区，确认内容真的变化了，再向用户汇报成功。

安全约束：

- 用户要求“替换选中内容”时，如果选区为空，必须先说明不能直接替换，并改走插入或只读建议。
- 用户要求“继续写”“润色这段”时，必须先读取当前文档或当前选区，禁止凭空续写。
- 如果读取失败或 runtime 返回不稳定，退回只读/诊断模式，不要伪造成功。

输出风格：

- 先简短说明你读到了什么，再说明准备如何写。
- 写入成功后，反馈一小段验证结果，不要只说“已完成”。
